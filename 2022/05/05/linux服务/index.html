4 docker镜像使用实例

可以参考：https://www.daimajiaoliu.com/daima/7b73b288d3b8809/docker

前提条件：

​ 文件夹Datmat：Dockerfile

​ 文件夹DockerBase-Official：Dockerfile

步骤一：首先将需要的.py文件全部放在对应的Dockerfile文件夹里面

步骤二：开启docker

systemctl start docker

步骤三：启动基础docker【DockerBase-Official】，这一步的目的是将通用的环境加载进去

cd DockerBase-Official
docker build -t docker_base-official .

步骤四：根据当前目录的Dockerfile内容构建名为data2mat的镜像，命令中最后的.表示构建的目录路径。

docker build -t data2mat .

步骤五： 运行Docker容器的命令是docker run，这条命令的基础是引用的镜像必须已经构建好。

docker run的逻辑先搜索本地是否有构建过这个镜像，找不到时会从远端服务器中的仓库拉取最新镜像回来。

运行docker

使用3个cpu运行
docker run --cpus 3 -v /etc/localtime:/etc/localtime -v /home/Docker/logs/log_test_dat2mat:/log -v /home/Docker/matFile:/usr/src/app/matFile -v /temp/ai-ecg/data:/usr/src/app/datFile -v /home/Docker/result:/usr/src/app/result -v /home/Docker/rFile:/usr/src/app/rFile --name v0.0.0.1_main_dat2mat -d v0.0.0.1_main_dat2mat

或者
使用所有的gpu运行，-v 是地址映射，将本机地址映射到docker地址内，冒号：前后分别是本机地址和docker地址
--name 为容器指定名字
--d 在后台运行容器并打印容器 ID
docker run --gpus all -v /etc/localtime:/etc/localtime -v /home/Docker/logs/log_test_dl:/log -v /home/Docker/matFile:/usr/src/app/matFile -v /home/Docker/datFile:/usr/src/app/datFile -v /home/Docker/result:/usr/src/app/result -v /home/Docker/result_dl1:/usr/src/app/result_dl1 -v /home/Docker/result_dl2:/usr/src/app/result_dl2 -v /home/Docker/rFile:/usr/src/app/rFile --name v0.0.0.1_main_dl -d v0.0.0.1_main_dl

其他操作：打开docker的环境

docker run -it docker_base-official bash
