1 安装软件

首先要安装python的集成开发环境。conda是跨平台包管理软件，2012年发布，是在数据分析界和Numpy和Scipy一样受欢迎的python工具。conda更多细节可以google。一些conda和anaconda的误解在下面文章可以了解。

http://nooverfit.com/wp/%E5%85%B3%E4%BA%8Econda%E5%92%8Canaconda%E4%B8%8D%E5%8F%AF%E4%B8%8D%E7%9F%A5%E7%9A%84%E4%BA%8B%E5%AE%9E%E5%92%8C%E8%AF%AF%E8%A7%A3-conda%E5%BF%85%E7%9F%A5%E5%BF%85%E4%BC%9A/

1.1 win环境下安装anaconda

安装conda，win环境下可以去官网直接下载并安装

https://www.anaconda.com/products/individual

下载完成双击安装，一路确认。安装完成的anaconda包含 jupyter notebook，Spyder

1.2 linux环境安装anaconda

  1. 通过 wget 下载 anaconda 安装包
wget https://repo.continuum.io/archive/Anaconda3-2018.12-Linux-x86_64.sh
  1. 利用 bash 命令安装到 anaconda 文件夹下
bash Anaconda3-2018.12-Linux-x86_64.sh -p anaconda/ -u

  1. 一直按yes，包括添加到环境

  2. 如果运行conda –version 显示没有此指令的话，需要把 anaconda 添加到变量中去，执行下面三句命令：

vim ~/.bashrc
  1. 然后将下面语句添加在./bashrc文件中，其中userNm为用户名字，需要修改为自己的用户名
echo 'export PATH="/home/userNm/anaconda/bin:$PATH"'
  1. 更新配置文件
source ~/.bashrc

1.3 Linux环境卸载anaconda

1）删除整个anaconda目录：
由于Anaconda的安装文件都包含在一个目录中，所以直接将该目录删除即可。

rm -rf anaconda

2）清理.bashrc中的Anaconda路径：

vim ~/.bashrc

注释掉之前添加的路径(或直接删除)： #export PATH=/home/userNm/anaconda3/bin:$PATH
3）更新配置文件，使其生效

source ~/.bashrc

2 python虚拟环境

linux和win均适配。打开命令窗口，查看已有的虚拟环境

conda env list

2.1 创建环境

env_name 为自定义环境名字，python=3.8为自定义的python的版本号

conda create -n env_name python=3.8

2.2 激活环境变量

conda activate env_name

2.3 关闭环境变量

conda deactivate

2.4 删除虚拟环境

conda remove -n env_name --all

3 jupyter notebook

3.1 修改默认路径

1)、修改配置文件

打开控制窗，输入下面命令，找到 jupyter_notebook_config.py 的位置

jupyter notebook --generate-config

打开 jupyter_notebook_config.py文件，找到 #c.NotebookApp.notebook_dir = ‘’ ，将自己的路径填进去，如下：

c.NotebookApp.notebook_dir = 'E:\\Proton\\Git\\JupyterML'

2)、修改属性的目标信息，将最后双引号那一串字母删掉，然后将自己的路径添加在后面

3)、修改默认打开方式为 “以管理员方式打开”。属性–>快捷方式–>高级–>用管理员身份运行–>确定

3.2 添加虚拟环境

首先输入下面命令，安装 ipykernel

pip install --user ipykernel
# 只能用pip，conda 会出错

然后将环境写入notebook的kernel中，其中env_name为环境名字

python -m ipykernel install --user --name=env_name

3.3 移除jupyter Notebook的内核

jupyter kernelspec remove env_name

7 Python 命令行参数的3种传入方式

参考链接：https://tendcode.com/article/python-shell/

7.1 sys.argv 模块

这个模块是我在工作中最常见的其他人写的 Python 脚本传入命令行参数的方式，也是最简单（粗暴）的方式。

# -*- coding:utf-8 -*-
# @Date  : 2021/12/10

import sys

def test_for_sys(year, name, body):
    print('the year is', year)
    print('the name is', name)
    print('the body is', body)

if __name__ == '__main__':
    try:
        year, name, body = sys.argv[1:4]
        test_for_sys(year, name, body)
    except Exception as e:
        print(sys.argv)
        print(e)

很好理解，由于 sys.argv 是一个由参数组成的列表，所以如果脚本中需要的参数比你命令行中输入的多，那肯定会报错，因为你输入的参数不够，反过来，如果你输入的参数比函数需要的多，那么无所谓，多的参数因为不会被提取使用，所以不影响脚本运行。

小结：sys.argv 形式传入参数的方式比较简单，但是也很死板，因为传入的参数是一个有序的列表，所以在命令行中必须按照脚本规定的顺序去输入参数，这种方法比较适合脚本中需要的参数个数很少且参数固定的脚本。

7.2 argparse 模块

argparse 模块也是 Python 自带的一个命令行参数模块，这个模块才是真的为了命令行参数而生的模块，相较之下 sys.argv 只是碰巧可以用在命令行参数上面而已。

# -*- coding:utf-8 -*-
# @Date  : 2019/2/27

import argparse


def test_for_sys(year, name, body):
    print('the year is', year)
    print('the name is', name)
    print('the body is', body)


parser = argparse.ArgumentParser(description='Test for argparse')
parser.add_argument('--name', '-n', help='name 属性，非必要参数')
parser.add_argument('--year', '-y', help='year 属性，非必要参数，但是有默认值', default=2017)
parser.add_argument('--body', '-b', help='body 属性，必要参数', required=True)
args = parser.parse_args()

if __name__ == '__main__':
    try:
        test_for_sys(args.year, args.name, args.body)
    except Exception as e:
        print(e)

使用 argparse 模块的方式也挺简单的，首先需要构建一个参数实例，也就是代码中的

parser = argparse.ArgumentParser(description='Test for argparse')

这行代码就生成了一个命令行参数的对象，之后就可以给对象添加一些参数属性，最后只需要从属性从提取传入的参数进行使用即可。

上面的代码添加了3个参数，添加参数的前两个字段很容易理解，--name 和 -n 都可以用来在命令行中使用，都表示了参数 name，这样后面使用 parse_args() 方法获取到所有参数之后，就可以使用 args.name 这种形式来提取对应的参数。

9.1 WIN环境：

进入文件”C:\Users\XHS”中，新建文件夹pip，在pip文件夹中新建文件pip.ini，打开该文件，写入内容

[global]
index-url = https://pypi.mirrors.ustc.edu.cn/simple/
pip3 config set global.index-url https://pypi.douban.com/simple/

查看使用的镜像源

pip3 config list

9.2 Linux环境：

mkdir ~/.pip
cd ~/.pip
vi pip.conf

9.3 conda环境

conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/

conda config --set show_channel_urls yes

此时，目录 C:\Users<你的用户名> 下就会生成配置文件.condarc，内容如下：

channels:
  - https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/
  - defaults

删除上述配置文件 .condarc 中的第三行，然后保存，最终版本文件如下：

channels:
  - https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/

通过命令 conda info 查看当前配置信息
