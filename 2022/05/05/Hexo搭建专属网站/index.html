Hexo搭建专属网站

2022-05-05 研究笔记 评论

0、前言

本文主要介绍利用hexo搭建自己的静态博客网站。首先简单介绍hexo安装，然后介绍本地搭建静态网站，最后介绍将网站部署在github。

官网：https://hexo.io/zh-cn/docs/

1、安装hexo框架

1.1、安装Node.js

直接到官网上下载安装即可，网址： https://nodejs.org/en/download/

Node.js 自带npm，npm类似于python的pip

1.2 安装hexo

npm install -g hexo-cli
hexo -v  #若安装成功，会出现hexo的版本号

2、搭建本地搭建Hexo

2.1 基础版(localhost:4000)

1）在博客的主目录栏输入cmd，打开命令窗

2）初始化一个网站框架，名字为test

hexo init test  ## test为指定的博客名字

初始化成功后，在指定目录下会出现一系列文件，如下图

3)、部署到本地

cd test ##进入test目录
hexo g	##生成静态页面
hexo s	##在服务器运行

在浏览器输入该地址即可显示如下图片

2.2 配置网站

2.2.1 主题

1)、进入 https://hexo.io/themes/index.html 预览主题，选择喜欢的主题下载。下载方法为，点击主题，进入主题的git地址，复制地址，clone。以pure为例，在git里面执行以下命令。

git clone https://github.com/cofess/hexo-theme-pure.git themes/pure

2)、将下载的主题的名字，复制到主目录的配置文件_config.yml中

2.2.2 中文字体

1、首先进入主题的语言目录，查看主题支持的语言，目录：themes\pure\languages

2、选择一种语言，填写在主目录的配置文件_config.yml中即可

2.2.3 创建删除文章

hexo new 文件名   #创建文件
hexo d -g		 #删除文件，前提是先删除本地文件
hexo new page tags
hexo new page categories

2.2.4 加载图片

1)、要添加图片，首先要npm install 一个hexo-asset-image的依赖

npm install https://github.com/CodeFalling/hexo-asset-image --save

显示下面的信息，就说明安装成功。安装成功后，node_modules文件夹会存在hexo-asset-image文件夹

2)、然后把_config.yml中的post_asset_folder设为true，这个配置的意思是每次new post一个博客，会增加一个和博客同名的文件夹。同名文件夹里面存储的是对应文件的图片资源

3)、Typora软件完美结合

进入 文件–>偏好设置–>图像–>复制到指定路径，然后将下面全部勾选上。

这样，插入的图片路径会映射为Hexo%一串数据。

然后依次执行下面命令 hexo clean，hexo g，hexo s。即可在http://localhost:4000/界面显示图片。

3、部署在git

1)、首先要有一个git账号，并新建一个仓库。需要注意的是，仓库名字最好为：用户名.github.io表示。例如用户名为test，则仓库名字最好为 test.github.io

2)、安装 hexo-deployer-git

npm install hexo-deployer-git --save

3)、修改配置文件

# Deployment
## Docs: https://hexo.io/docs/one-command-deployment
deploy:
    type: git
    repo: https://github.com/test/test.github.io.git
    branch: gh-pages

注意：branch选择 gh-pages。

4)、部署

hexo clean
hexo g
hexo d

5、Hexo部署过程中出现错误

Error: Spawn failed

解决办法

cd /usr/local/src/hexo/hanyubolg/   ##进入站点根目录
rm -rf .deploy_git/				   ##删除git提交内容文件夹
git config --global core.autocrlf false ##执行
hexo clean && hexo g && hexo d ##最后
